# 摘要

生成式人工智能大幅降低了图像伪造门槛,并带来两重叠加的安全风险。一方面,生成端的安全对齐机制可被"超监管"内容绕过——涉及战争、恐怖主义、枪械等敏感场景的高危图像能够绕过对齐被正常生成,并用于假新闻与谣言传播,威胁社会稳定\[9\];另一方面,这些内容与普通图像共享同一条社交媒体传播链,而平台的压缩、缩放与截图转存恰恰会削弱伪造证据。这意味着最需要被识别的高危伪造,恰恰要经过最能摧毁伪造证据的传播处理。因此,传播链末端的审核人员需要的不只是真/假标签,而是可复核的证据、风险分级与处置依据。

TraceGuard 面向社交媒体传播链末端的 AIGC 图像审核,构建集跨域真伪检测、可解释证据展示、篡改区域定位、多证据风险融合与检测报告导出于一体的图像安全审核平台。系统以去全局池化的 MambaOut 门控卷积骨干与多核最大均值差异(MK\-MMD)无监督域自适应为核心判别路线,以 Stage2 Grad\-CAM 热力图与 patch/特征混合定位提供并行证据,经五维风险融合输出低/中/高风险等级与人工复核建议,并以"全局判断、局部证据、融合结论"三层输出合同保证 Web、API、CLI 各入口结论一致。

测试覆盖八类生成器平衡盲测、8000 组同图三平台传播成对实验、确定性派生扰动拆解与真实浏览器闭环验收,使用 Accuracy、Macro F1、ROC AUC、Recall、成对概率变化与性能保持率评价。本作品的核心贡献在于**系统化量化了社交媒体传播对伪造证据的破坏**:同图成对协议显示 Facebook 条件下 Fake Recall 由 59.55% 降至 21.675%、平均伪造概率下降 0.3162;确定性派生扰动进一步定位 JPEG 量化为主要证据破坏者(jpeg75 保持率仅 16.65%、jpeg50 仅 3.93%),并以真图假阳检查证伪了 resize 带来的表面鲁棒性假象。相较于既有检测工作多止于原始图像或单次 JPEG 压缩的评价口径,本作品在传播链末端真实条件下给出了证据退化的系统度量,并对证据衰减样本给出人工复核分流方案。

关键词:AIGC 内容安全;伪造检测;跨域泛化;可解释取证;篡改定位;风险融合

# 第一章 作品概述

1\.1 背景与威胁

生成式人工智能显著降低了图像内容的生成与编辑门槛,虚假新闻配图、舆情误导、身份冒用、电子证据污染与平台内容审核压力随之上升。仅给出"真/假"标签的检测器难以满足实际安全审核需求,因为审核人员还需要知道模型依据、可疑区域、风险等级以及可复核的检测记录。

更严峻的威胁在于,图像内容安全不能只依赖生成端的安全对齐。已有研究表明,生成模型的对齐机制仅拦截显式触发预设范式的请求,涉及战争、恐怖主义、枪械等敏感场景的"超监管"图像可以绕过对齐被正常生成,且未被现有检测方法专门覆盖,一旦用于假新闻和谣言传播将威胁社会稳定\[9\]。此类高危内容与常规内容共享同一条社交媒体传播链路:上传、转码、压缩、缩放、截图转存都会作用其上。由此形成一个被现有研究忽视的悖论——最需要被识别的高危伪造,恰恰要经过最能摧毁伪造证据的传播处理。

因此,传播链末端的第三方审核成为不可或缺的兜底环节:它必须在跨生成器、跨数据集、跨图像风格,且经历压缩、缩放、裁剪、截图转存等真实传播扰动之后,仍能给出可解释、可复核的判断,并在证据不足时明确转人工复核而非给出过度自信的结论。TraceGuard 据此将参赛作品定位为信息安全场景下的 AIGC 图像安全审核平台,而不是单一模型演示,并把"真实传播环境中的跨域泛化与可解释取证"作为核心关切。

1\.2 相关工作与现有不足

现有 AIGC 图像检测研究可大致分为三条线。其一是**伪造痕迹检测线**:基于统计纹理、频域特征或生成器残留痕迹训练二分类器\[1\],此类方法在同域测试中表现良好,但在新生成模型、新分辨率、新压缩链路或二次传播场景中容易出现明显性能下降\[1\-2\]。其二是**通用检测线**:利用大规模预训练表征提升跨生成器泛化能力\[2\],但其评测通常止于原始图像,未覆盖真实传播链末端的重编码图像。其三是**可解释分析线**:以 Grad\-CAM\[7\] 为代表的归因可视化能够展示模型关注区域,但多数工作止步于展示,未与定位证据、风险分级和审核处置流程结合。

数据与评测层面存在两处缺位。其一,主流评测集(如 GenImage\[3\])主要覆盖动物、植物、车辆与自然风景等常规内容,对绕过生成端对齐机制的"超监管"高危内容,公开数据与针对性评测均不足\[9\],检测系统在此类内容上的行为边界缺乏系统评价。其二,已有针对压缩/传播鲁棒性的评价大多止于原始图像或单次 JPEG 压缩这类孤立扰动,缺少在真实社交平台传播链末端、以同图成对协议做的系统度量;而这恰恰是高危伪造进入审核视野时所处的真实工况。

综上,现有工作在跨域泛化、传播鲁棒性评测、可解释取证与审核闭环四个方面各有进展,但缺少将四者组织为同一平台闭环、并以传播链末端真实条件系统评测的作品。缺传播链末端的系统评测与审核闭环,正是 TraceGuard 的切入点。

1\.3 作品定位与应用场景

本作品面向开放式自主命题作品赛,核心场景是社交媒体谣言与高危假新闻配图筛查,并延伸至平台内容审核、新闻图片核验与电子证据初筛。其中,**超监管高危内容的假新闻配图核验是头号场景**:此类内容社会危害最大,且往往在经过平台压缩传播、伪造证据被削弱之后才进入审核视野。系统输出不仅包括真伪概率,还包括风险分数、可疑区域、解释文本和报告化结论,使检测结果能够被复核、沟通与归档。以下按三类工况走查系统价值。

**平台内容审核。** 内容平台每日面对海量用户上传图像,审核拿到的通常是经平台转码后的版本。TraceGuard 的风险等级可用于自动分流:低风险直接放行、高风险进入拦截队列、中风险与证据冲突样本转人工复核;热力图与可疑区域为复核人员提供直接依据,检测报告支持处置留痕。

**新闻图片核验。** 编辑在发稿前对来源不明的配图做单图核验:系统给出真伪概率、可疑区域与结构化解释,核验结论连同证据图导出为报告归档,可用于事后追溯。对涉及公共安全的高危配图(如战争、恐怖场景类内容\[9\]),在证据不足时明确提示人工复核,而非给出过度自信的结论。

**电子证据初筛。** 取证场景要求可追溯:TraceGuard 记录输入哈希、权重版本与全部证据产物,`label` 与 `tamper_type` 分栏呈现,全局与局部证据冲突时保留分歧并提示人工鉴定,避免单一分数误导初筛结论。

1\.4 特色描述与应用前景

本作品的特色集中在三点。第一,**证据合同**:全局判断(`label` 与 `fake_prob`)仅由检测器产生,局部证据并行输出、不得静默覆盖全局结论,冲突即提示人工复核,保证系统在证据退化时不给出过度自信的结论。第二,**传播链系统评测**:以同图成对协议系统量化社交媒体传播对检测证据的破坏,并用确定性派生扰动拆解"是哪种处理在摧毁证据",使性能主张与真实审核工况对齐;这一评测维度是本作品区别于既有工作的核心贡献。第三,**工程闭环**:Web、API、CLI 与批量入口复用同一分析管线,单张消费级 GPU 即可部署(固定样例中位延迟 0.520 秒),检测结论可直接导出为结构化报告。

应用前景方面,系统可作为内容平台审核流水线的前置风险分流组件、新闻与政务机构的图像核验工具,以及电子证据初筛的辅助手段。面向绕过生成端对齐的超监管高危内容监测\[9\],是本作品后续扩展的重点方向——该方向的概念与数据来自公开文献,本作品目前仅就"评测自研检测器在此类场景下的行为边界"展开,不涉及高危内容的生成或训练。

1\.5 团队分工与阶段计划

团队三名成员按模块分工:成员 A 负责系统集成、平台工程、报告与演示;成员 B 负责跨域检测模块与检测实验;成员 C 负责可解释分析、篡改定位与风险融合评价。跨模块接口以数据合同形式约定,任何一方不得单方面变更响应字段的语义,以保证四个入口结论一致。

研制计划:6 月下旬完成报名、定题与接口确认;7 月上旬完成最小可演示系统;7 月中旬完成跨域盲测、社交媒体传播鲁棒性与案例实验;7 月下旬完成报告与答辩材料并封版提交。

# 第四章 创新性说明

4\.1 面向真实传播环境的跨域检测叙事

作品以跨生成器和跨平台传播作为核心测试维度,面向来源复杂、处理链路多样的网络图像审核需求。在生成端安全对齐可被绕过的现实约束下\[9\],传播链末端的第三方检测承担兜底职责,这是本作品聚焦传播后图像审核能力的根本动机。

**技术创新点 1:去全局池化的门控卷积骨干。** 传统骨干网络末端的全局平均池化会不可逆地压缩空间伪影信息。本作品选取 MambaOut 门控 CNN 并针对 AIGC 图像检测适配:移除末端全局平均池化,将最后一阶段特征保留为 2×2 空间网格并展平为 2304 维密集特征,再经瓶颈层降至 256 维,在保留微观伪影空间结构的同时控制后续域对齐的计算维度。与直接使用 ResNet/ViT 全局特征的常规做法\[1\-2\]相比,该设计以保留空间证据为首要目标。

**技术创新点 2:多核 MMD 的无监督域自适应机制。** 引入 5 核高斯 MK\-MMD,在再生核希尔伯特空间对齐源域与目标域的特征分布,目标域无需任何标注。与仅靠数据增广提升泛化的做法相比,该机制显式建模并度量多尺度的域间分布差异。

**技术创新点 3:轻量化设计与部署可行性。** MambaOut\-Small 骨干约 45.44M 参数,已在单张 RTX 4060 Laptop GPU 上完成真实 Web 闭环:固定 BigGAN 样例的单图 API 分析耗时约 0.56 秒;12 请求、并发度 3 的短时并发烟测 12/12 成功,中位延迟 0.520 秒、P95 2.075 秒(表 3.7),支持单卡审核工作台部署。

4\.2 社交媒体传播链末端的系统化鲁棒性评测

本作品把"传播链末端的证据退化"从一个附带观察升格为可复现的系统评测,这是区别于既有检测工作的核心创新点。评测以同图成对协议为基础:对固定的 8000 张 GenImage fake 图像,分别运行 Original、Facebook、WeChat、Weibo 四种条件,以同一 `sample_id` 追踪同一图像在传播前后的 `fake_prob` 与检出变化,从而直接度量"证据被削弱了多少",而非仅仅比较两个不同数据集上的分类精度。结果显示 Facebook 条件下 Fake Recall 由 59.55% 降至 21.675%、平均伪造概率下降 0.3162(表 3.2),且逐生成器保持率差异悬殊(BigGAN、VQDM、ADM、Glide 仅 2.3%--11.8%,Midjourney、Wukong 达 63.5%--78.7%),说明传播鲁棒性必须同时看绝对检出率与相对保持率。

在此基础上,本作品用**确定性派生扰动**进一步拆解"是哪种处理在摧毁证据":对全量样本施加 jpeg75、jpeg50、resize50、screenshot 五条件,定位出 JPEG 量化是主要的证据破坏者(jpeg75 保持率仅 16.65%、jpeg50 仅 3.93%,表 3.4),与 Facebook 传播退化互证(Facebook 链路含强 JPEG 重压缩)。针对 resize50 表面上"保持率升至 137.94%"的反常,进一步用真图假阳检查证伪:resize50 把 1000 张真图的假阳率从 1.50% 抬到 17.50%(表 3.5),证明"降采样再放大"被检测器普遍读作生成伪影、不分真假地整体抬高 fake 预测,因此这并非鲁棒性收益,不作为传播鲁棒性的正面证据。相较于既有工作多止于原始图像或单次 JPEG 压缩这类孤立扰动的评测口径,本作品在真实社交平台传播链末端给出了证据退化的系统度量与成因拆解,使系统的性能主张与真实审核工况对齐,并直接支撑"证据退化即转人工复核"的处置策略。

4\.3 可解释取证与报告化闭环

系统将检测分数、热力图、可疑区域、解释文本与报告导出整合于同一流程(证据见图 3.3 与表 3.7),使检测结果可复核、可沟通、可归档。与仅输出分类分数的检测器\[1\-2\]和仅做归因可视化的方法\[7\]相比,本作品将解释产物纳入审核处置链路,提升了 AIGC 检测从算法结果到安全处置证据的可用性。

4\.4 多证据风险融合

系统通过真伪概率、热力图、篡改定位与文件处理信息的五维加权融合(见 2.4.5),输出统一的低/中/高风险等级与报告化解释,减少单一模型分数在复杂场景中的误导。在 Facebook 派生平衡集的分层留出校准上,人工复核策略的留出集 F1 达 0.9877、高风险边界 F1 达 1.0(见 3.4),说明融合分数可支撑审核分流。融合结论不改写 Detector 的全局标签。

4\.5 平台化闭环与工程可复用

TraceGuard 采用模块化工程结构,四个入口(Web UI、API、CLI 与批量分析)复用同一 ExplanationPipeline 与统一响应合同,将跨域检测、解释展示、篡改定位与报告导出组织为同一平台闭环,避免不同入口产生相互矛盾的真伪结论;179 项自动化回归保障各入口行为一致性。

上述各机制的分项量化消融(骨干去全局池化改造与 MK\-MMD 无监督域自适应的独立贡献)列入后续工作。

# 第五章 总结

TraceGuard 面向 AIGC 图像安全中的检测、解释、取证与报告需求,构建跨域检测、可解释分析、篡改定位、风险融合与平台展示一体化的作品框架。其叙事主线是:超监管高危内容是危害最大的伪造,却与普通内容共享同一条最能摧毁伪造证据的社交媒体传播链,因此传播链末端需要可解释、传播鲁棒、证据退化即转人工的第三方审核。

**已完成进展**:系统已实现 MambaOut\-Small 全局检测、Stage2 Grad\-CAM、patch 与特征统计融合定位、五维风险评分、中文解释以及 Web/API/CLI/HTML 报告入口,并以 179 项自动化回归保障入口行为一致。当前权重已完成八生成器平衡盲测、三类社交平台传播成对实验与确定性派生扰动实验。作为核心贡献,传播前后成对实验系统量化了 Facebook 压缩导致的显著性能退化(Fake Recall 由 59.55% 降至 21.675%),派生扰动进一步定位 JPEG 量化为主要证据破坏者(jpeg75 保持率 16.65%),并以真图假阳检查证伪了 resize 的表面鲁棒性假象;这些结果共同表明,不能仅用类别不平衡平台集上的较高 Accuracy 代替传播链末端的鲁棒性分析。

系统最终保持"全局判断、局部证据、融合结论"三层输出。全局 `label` 与 `fake_prob` 仅由 Detector 产生;热力图与局部定位作为并行证据分支;风险模块依据多维证据给出审核优先级与人工复核建议。当前来源的定位边界评价与留出风险校准已写入报告;定位分支的低像素重合、clean 误报与单一传播来源限制了其外推范围,而这恰恰印证了系统的核心处置主张——在证据退化、来源单一或证据冲突时明确转人工复核,而不给出过度自信的结论。骨干与域自适应机制的分项消融列为后续工作。
