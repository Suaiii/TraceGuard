# 摘要

TraceGuard 是一个面向社交媒体传播链末端的可解释 AIGC 图像审核平台。它的能力不止是"一个检测器",而是"一个在证据退化时依然可信的审核平台":在最危险的伪造场景上零样本识别、在证据被压缩削弱时能够识别退化并转入人工复核、且能在消费级硬件上部署运行。面对社会危害最大的超监管高危内容(战争、军事、武器类场景),检测器在从未见过该域数据的零样本条件下,对 9 个生成器的 Fake Recall 均落在 94.0%–100.0%、总体达 98.84%,而真图假阳率仅 1.4%——高召回并未以牺牲真图判别为代价,其 Real Recall 在跨生成器盲测中稳定保持 99.80%。系统以"全局判断、局部证据、融合结论"三层输出合同保证证据退化时不给出过度自信的结论,并在单张消费级 GPU(RTX 4060 Laptop)上完成真实 Web 闭环,固定样例中位延迟 0.520 秒。

上述能力对应一个被现有研究忽视的现实威胁。生成式人工智能大幅降低了图像伪造门槛,并带来两重叠加的安全风险。一方面,生成端的安全对齐机制可被"超监管"内容绕过——涉及战争、恐怖主义、枪械等敏感场景的高危图像能够绕过对齐被正常生成,并用于假新闻与谣言传播,威胁社会稳定\[9\];另一方面,这些内容与普通图像共享同一条社交媒体传播链,而平台的压缩、缩放与截图转存恰恰会削弱伪造证据。这意味着最需要被识别的高危伪造,恰恰要经过最能摧毁伪造证据的传播处理。因此,传播链末端的审核人员需要的不只是真/假标签,而是可复核的证据、风险分级与处置依据。

TraceGuard 面向社交媒体传播链末端的 AIGC 图像审核,构建集跨域真伪检测、可解释证据展示、篡改区域定位、多证据风险融合与检测报告导出于一体的图像安全审核平台。系统以去全局池化的 MambaOut 门控卷积骨干与多核最大均值差异(MK\-MMD)无监督域自适应为核心判别路线,以 Stage2 Grad\-CAM 热力图与 patch/特征混合定位提供并行证据,经五维风险融合输出低/中/高风险等级与人工复核建议,并以"全局判断、局部证据、融合结论"三层输出合同保证 Web、API、CLI 各入口结论一致。

测试覆盖八类生成器平衡盲测、8000 组同图三平台传播成对实验、确定性派生扰动拆解与真实浏览器闭环验收,使用 Accuracy、Macro F1、ROC AUC、Recall、成对概率变化与性能保持率评价。本作品的核心贡献是双重的:一方面在头号高危场景上验证了检测器的零样本识别力、真图判别力与三层合同的可信路由能力;另一方面以传播链末端的系统评测诚实标定这些能力的行为边界。同图成对协议显示 Facebook 条件下 Fake Recall 由 59.55% 降至 21.675%、平均伪造概率下降 0.3162;确定性派生扰动进一步定位 JPEG 量化为主要证据破坏者(jpeg75 保持率仅 16.65%、jpeg50 仅 3.93%),并以真图假阳检查证伪了 resize 带来的表面鲁棒性假象。这一证据退化的系统度量为平台的自我诊断提供依据:系统能够识别证据何时不可信,并据此将可疑样本分流至人工复核。相较于既有检测工作多止于原始图像或单次 JPEG 压缩的评价口径,本作品在传播链末端真实条件下给出了证据退化的系统度量,并对证据衰减样本给出人工复核分流方案。

关键词:AIGC 内容安全;伪造检测;跨域泛化;可解释取证;篡改定位;风险融合

# 第一章 作品概述

1\.1 背景与威胁

生成式人工智能显著降低了图像内容的生成与编辑门槛,虚假新闻配图、舆情误导、身份冒用、电子证据污染与平台内容审核压力随之上升。仅给出"真/假"标签的检测器难以满足实际安全审核需求,因为审核人员还需要知道模型依据、可疑区域、风险等级以及可复核的检测记录。

更严峻的威胁在于,图像内容安全不能只依赖生成端的安全对齐。已有研究表明,生成模型的对齐机制仅拦截显式触发预设范式的请求,涉及战争、恐怖主义、枪械等敏感场景的"超监管"图像可以绕过对齐被正常生成,且未被现有检测方法专门覆盖,一旦用于假新闻和谣言传播将威胁社会稳定\[9\]。此类高危内容与常规内容共享同一条社交媒体传播链路:上传、转码、压缩、缩放、截图转存都会作用其上。由此形成一个被现有研究忽视的悖论——最需要被识别的高危伪造,恰恰要经过最能摧毁伪造证据的传播处理。

因此,传播链末端的第三方审核成为不可或缺的兜底环节:它必须在跨生成器、跨数据集、跨图像风格,且经历压缩、缩放、裁剪、截图转存等真实传播扰动之后,仍能给出可解释、可复核的判断,并在证据不足时明确转人工复核而非给出过度自信的结论。TraceGuard 据此将参赛作品定位为信息安全场景下的 AIGC 图像安全审核平台,而不是单一模型演示,并把"真实传播环境中的跨域泛化与可解释取证"作为核心关切。

1\.2 相关工作与现有不足

现有 AIGC 图像检测研究可大致分为三条线。其一是**伪造痕迹检测线**:基于统计纹理、频域特征或生成器残留痕迹训练二分类器\[1\],此类方法在同域测试中表现良好,但在新生成模型、新分辨率、新压缩链路或二次传播场景中容易出现明显性能下降\[1\-2\]。其二是**通用检测线**:利用大规模预训练表征提升跨生成器泛化能力\[2\],但其评测通常止于原始图像,未覆盖真实传播链末端的重编码图像。其三是**可解释分析线**:以 Grad\-CAM\[7\] 为代表的归因可视化能够展示模型关注区域,但多数工作止步于展示,未与定位证据、风险分级和审核处置流程结合。

数据与评测层面存在两处缺位。其一,主流评测集(如 GenImage\[3\])主要覆盖动物、植物、车辆与自然风景等常规内容,对绕过生成端对齐机制的"超监管"高危内容,公开数据与针对性评测均不足\[9\],检测系统在此类内容上的行为边界缺乏系统评价。其二,已有针对压缩/传播鲁棒性的评价大多止于原始图像或单次 JPEG 压缩这类孤立扰动,缺少在真实社交平台传播链末端、以同图成对协议做的系统度量;而这恰恰是高危伪造进入审核视野时所处的真实工况。

综上,现有工作在跨域泛化、传播鲁棒性评测、可解释取证与审核闭环四个方面各有进展,但缺少将四者组织为同一平台闭环、并以传播链末端真实条件系统评测的作品。缺传播链末端的系统评测与审核闭环,正是 TraceGuard 的切入点。

1\.3 作品定位与应用场景

本作品面向开放式自主命题作品赛,核心场景是社交媒体谣言与高危假新闻配图筛查,并延伸至平台内容审核、新闻图片核验与电子证据初筛。其中,**超监管高危内容的假新闻配图核验是头号场景**:此类内容社会危害最大,且往往在经过平台压缩传播、伪造证据被削弱之后才进入审核视野。而 TraceGuard 在这一头号场景上具备可量化的识别能力:在从未见过该域数据的零样本条件下,检测器对 9 个超监管生成器的总体 Fake Recall 达 98.84%、真图假阳率仅 1.4%(表 3\.6),说明其学到的生成器伪影判据能够跨内容域迁移到最高危的内容上。系统输出不仅包括真伪概率,还包括风险分数、可疑区域、解释文本和报告化结论,使检测结果能够被复核、沟通与归档。以下按三类工况走查系统价值。

**平台内容审核。** 内容平台每日面对海量用户上传图像,审核拿到的通常是经平台转码后的版本。TraceGuard 的风险等级可用于自动分流:低风险直接放行、高风险进入拦截队列、中风险与证据冲突样本转人工复核;热力图与可疑区域为复核人员提供直接依据,检测报告支持处置留痕。

**新闻图片核验。** 编辑在发稿前对来源不明的配图做单图核验:系统给出真伪概率、可疑区域与结构化解释,核验结论连同证据图导出为报告归档,可用于事后追溯。对涉及公共安全的高危配图(如战争、恐怖场景类内容\[9\]),在证据不足时明确提示人工复核,而非给出过度自信的结论。

**电子证据初筛。** 取证场景要求可追溯:TraceGuard 记录输入哈希、权重版本与全部证据产物,`label` 与 `tamper_type` 分栏呈现,全局与局部证据冲突时保留分歧并提示人工鉴定,避免单一分数误导初筛结论。

1\.4 特色描述与应用前景

本作品的特色集中在三点产品能力,并以传播链系统评测作为诚实背书。

第一,**零样本超监管识别与真图判别**:面向社会危害最大的超监管高危内容,检测器在从未见过该域数据的零样本条件下,对 9 个生成器的总体 Fake Recall 达 98.84%、真图假阳率仅 1.4%(表 3\.6),且跨生成器盲测的 Real Recall 稳定保持 99.80%(表 3\.1)——高召回未以牺牲真图判别为代价;其识别能力可从通用内容域迁移到高危内容域。

第二,**可信路由的证据合同**:全局判断(`label` 与 `fake_prob`)仅由检测器产生,局部证据并行输出、不得静默覆盖全局结论,冲突即提示人工复核,保证系统在证据退化时不给出过度自信的结论;当证据因传播压缩而失真时,系统能据此触发复核而非静默改判。

第三,**工程闭环与单卡部署**:Web、API、CLI 与批量入口复用同一分析管线,单张消费级 GPU 即可部署(固定样例中位延迟 0.520 秒),检测结论可直接导出为结构化报告。

在上述能力之上,本作品以传播链系统评测标定其行为边界:以同图成对协议系统量化社交媒体传播对检测证据的破坏,并用确定性派生扰动拆解"是哪种处理在摧毁证据",使性能主张与真实审核工况对齐。这一评测维度同时体现平台的自我诊断能力——系统能够识别证据何时退化,并据此将可疑样本分流至人工复核。

应用前景方面,系统可作为内容平台审核流水线的前置风险分流组件、新闻与政务机构的图像核验工具,以及电子证据初筛的辅助手段。面向绕过生成端对齐的超监管高危内容监测\[9\],是本作品后续扩展的重点方向——该方向的概念与数据来自公开文献,本作品目前仅就"评测自研检测器在此类场景下的行为边界"展开,不涉及高危内容的生成或训练。

# 第二章 作品设计与实现

2\.1 总体架构

TraceGuard 采用模块化流水线设计。系统接收单张输入图像并完成预处理后，由跨域 AIGC 检测模块给出全局真伪判断；在此基础上，可解释性分析模块与篡改（可疑区域）定位模块并行产出证据，多证据风险融合模块据此形成风险分级；最终结果经 Web 界面可视化展示，并导出结构化检测报告。检测、解释、定位与报告导出被组织为可演示、可测试、可复核的图像安全审核闭环。

图 2\-1 · TraceGuard 系统总体架构。`Detector.predict()` 是 `label` 与 `fake_prob` 的唯一来源；Grad\-CAM 与局部定位作为并行证据分支；风险和文本模块将各来源组织为全局判断、局部证据与融合结论。

2\.2 跨域 AIGC 图像检测模块

跨域检测模块负责对输入图片输出真伪概率和类别标签，是 TraceGuard 的核心判别引擎。由于真实网络传播中 AIGC 图像的来源模型多样，训练域（源域）与测试域（目标域）之间存在显著分布差异，传统单一域训练的检测器在跨生成器、跨数据集场景中容易出现性能下降\[2-3\]。为此，本模块以门控卷积骨干提取判别特征，并引入多核最大均值差异（MK\-MMD）无监督域自适应对齐源域与目标域的特征分布；其跨域贡献仍需消融原始表支持，本报告不提前写成已验证提升。

2\.2\.1 骨干网络设计：MambaOut\-Small 去全局池化

传统 AIGC 检测方法普遍采用 ResNet 或 ViT 等骨干网络，这些网络末端通常包含全局平均池化（Global Average Pooling, GAP）层，将空间特征图压缩为单一特征向量。然而，GAP 的全局平均操作在提取语义信息的同时，会不可逆地破坏 AIGC 图像中关键的微观伪影——例如生成器在像素级留下的网格模式、异常纹理和局部色偏。这些细粒度痕迹恰恰是区分真实图像与 AI 生成图像的重要依据。

本作品选取 MambaOut\-Small 作为骨干网络。MambaOut 是一种基于门控卷积（Gated CNN）的视觉架构，其研究表明部分视觉任务并不必然需要状态空间模型\[4\]。MambaOut\-Small 的具体配置为：4 个阶段，深度 \[3, 4, 27, 3\]，各阶段特征维度 \[96, 192, 384, 576\]，总参数量约 44\.8M（不含新增层）。

核心改进：移除原始 MambaOut 的 GAP 层，将最后一阶段输出的 \(B, 7, 7, 576\) 特征图经 LayerNorm 后进行自适应池化至 \(2, 2\)，展平为 2304 维密集特征向量，完整保留像素级空间结构信息。随后通过瓶颈层（Linear\(2304, 256\) → BatchNorm1d → ReLU）将特征降维至 256 维，在保留判别信息的同时避免后续 MMD 计算遭遇维度灾难。最后接入监督分类头 Linear\(256, 2\) 输出 real/fake 二分类 logits。

2\.2\.2 无监督域自适应：多核最大均值差异（MK\-MMD）

在无监督域自适应（UDA）设定下，目标域数据无任何类别标注。本模块引入多核最大均值差异（Multi\-Kernel Maximum Mean Discrepancy, MK\-MMD）作为分布对齐正则项\[5\]，在训练阶段将源域和目标域特征映射到再生核希尔伯特空间（RKHS），通过最小化两域在此空间中的经验 MMD 距离学习域不变特征表示。

MK\-MMD 采用 5 个不同带宽的高斯核进行线性组合：

$ k(x, y) = \sum_{u=1}^{5} \exp\left(-\frac{|x - y|^2}{2\sigma_u^2}\right)$

其中 \($\sigma_u$\) 采用几何级数分布：\($\sigma_u = 2^{u-3} \cdot \bar{d}$\)，\($\bar{d}$\) 为批次内样本对的平均欧氏距离。5 个核分别捕获不同尺度的分布差异，使得域对齐对源域与目标域之间的复杂分布偏移具有更强的适应能力。

经验 MMD 距离的计算采用 V\-统计量（V\-statistic）估计器：

$ \widehat{MMD}^2 = \frac{1}{n_s^2}\sum_{i,j} k(x_i^s, x_j^s) + \frac{1}{n_t^2}\sum_{i,j} k(x_i^t, x_j^t) - \frac{2}{n_s n_t}\sum_{i,j} k(x_i^s, x_j^t)$

该实现参考 Transfer\-Learning\-Library 中的 DAN（Deep Adaptation Networks）模块\[6\]，并根据本任务的数据规模和模型结构进行了适配。

2\.2\.3 联合训练策略

训练阶段采用双流（Two\-Stream）输入架构：每个训练 batch 同时送入源域图像（含 real/fake 标签）和目标域图像（无标签）。训练目标为联合损失函数：

$ \mathcal{L}_{total} = \mathcal{L}_{cls} + \beta \cdot \mathcal{L}_{MMD} $

其中 $\mathcal{L}_{cls}$ 为源域交叉熵损失（带标签平滑 0\.1），$\mathcal{L}_{MMD}$ 为源域与目标域 256 维瓶颈特征的 MK\-MMD 距离。平衡因子 \($\beta$\) 采用渐进调度策略：训练初期设 \($\beta$ = 0\) 让分类器充分学习源域判别特征；在训练过程中 \($\beta$\) 线性增长至目标值 \($\beta_{max} = 1.0$\)，逐步增加域对齐的强度。

此外，训练策略还包括以下优化组件：

- **差分学习率**：ImageNet 预训练的骨干网络使用较低学习率$(1\times10^{-5})$，新增的瓶颈层和分类头使用较高学习率$(1\times10^{-3})$，以防止预训练知识被过度扰动；

- **学习率 Warmup**：前 5 个 epoch 学习率从 0 线性升温至目标值，避免训练初期因随机梯度更新破坏预训练权重；

- **余弦退火调度**：Warmup 结束后采用余弦退火将学习率衰减至初始值的 1%；

- **梯度裁剪**：设置最大梯度范数为 5\.0，防止训练不稳定。

2\.2\.4 模块接口定义

模块对外提供标准化 REST API 接口，输入输出字段定义如下：

|字段|类型|含义|
|---|---|---|
|`label`|string|全局判定 `real`/`fake`，仅由 Detector 产生|
|`fake_prob`|float|伪造概率，取值 0~1|
|`tamper_type`|string|局部证据类型：`confirmed_real`/`local_tamper`/`full_aigc`/`full_aigc_hotspots`|

2\.3 可解释检测与取证解释模块

可解释模块的数据源头为图像检测主模块，核心作用是面向审核人员说明模型判断依据和证据分歧，输出热力图、可疑目标区域标注、局部异常提示以及标准化文字说明。

2\.3\.1 Grad\-CAM 热力图生成

早期版本通过瓶颈层权重将 256 维特征反向归因到 2×2 空间格点。该方法存在根本性局限，2×2 格点无法实现"精准高亮 AIGC 网格、局部色偏、扩散残差痕迹"的要求，上采样后的热力图本质是 4 个色块的模糊扩散，属于典型的"装饰性热力图"。

升级方案在 MambaOut backbone 的 stage2 输出（384 通道 × 14×14 = 196 个空间格点）上实施 Grad\-CAM\[7\]。注册 forward hook 捕获激活 $A ∈ R^{384×14×14}$，同时注册 backward hook 捕获梯度 $G ∈ R^{384×14×14}$。对 fake 类 logit 反向传播，取梯度空间均值作为通道权重，计算类激活图得到 \[14, 14\] 的热力矩阵：

$w_c = \frac{1}{Z}\sum_{i,j} G_{c,i,j}, \quad L_{\text{CAM}} = \text{ReLU}\left(\sum_{c} w_c A_c\right)$

其中$A_c \in \mathbb{R}^{14 \times 14}$为第 $c$个通道的激活图，$G_c$为对应梯度，$Z = 14 \times 14$，再上采样到原图尺寸，并将数值归一化到 \[0, 1\]。空间格点从 4 个提升到 196 个，为展示模型对局部纹理和残差模式的响应提供更细粒度的分类证据，但不把该响应解释为像素级篡改真值。

2\.3\.2 热力图后处理与可视化

Grad\-CAM 输出的热力图首先经过高斯平滑（σ=3\.0），消除 14×14 上采样到原图尺寸时产生的棋盘格伪影。之后通过自定义 Colormap 映射为彩色图像——采用 7 个颜色锚点分段线性插值：

深蓝\(0\.0, 低伪造可疑\) → 青\(0\.4\) → 绿\(0\.55\) → 黄\(0\.7\) → 橙红\(0\.85\) → 红紫\(1\.0, 极高伪造可疑\)。

最终输出两种图像格式：原图与热力图半透明叠加的 `overlay`（α=0\.5，供前端渲染和 PDF 报告），以及纯热力掩膜 `mask`（供取证分析）。前者辅助快速定性判断，后者保留精确的热力分数分布供定量分析。

![检测结果与可解释证据示例](../docs/figures/system/detection_example.svg)

图 2\-2 · 检测结果与可解释证据示例。a，输入图像；b，Stage2 Grad\-CAM 叠加图；c，局部定位分支给出的可疑区域。热力响应与红框来自不同解释路径，两者仅仅用于指出模型关注和局部异常位置，不等同于像素级篡改真值。

2\.3\.3 自然语言解释生成

采用三段式模板生成标准化中文解释文本。解释内容根据篡改分类结果自动匹配对应描述——确认真实、局部篡改、全图 AIGC 生成、全图 AIGC 含热点区域四种情况各有独立的结论措辞：

【总体结论】判定结果 \+ 篡改类型 \+ 置信度 \+ 风险等级 \+ 处置建议
【取证分析】热力图伪影特征描述（强烈/中等/微弱三档）\+ 响应统计量
【定位详情】逐区域坐标\(x,y,w,h\) \+ 面积\(px\) \+ patch 级风险分 \+ 复核建议

解释文本随检测结果自动生成，可直接嵌入 HTML 报告或导出为纯文本。系统将全局标签 `label`（`real` 或 `fake`）与局部证据类型 `tamper_type` 分栏展示：前者仅由 Detector 产生，后者描述定位分支是否发现异常区域。两类证据不一致时，报告保留分歧并给出人工复核提示，不由局部分支静默改写真伪结论。

2\.4 篡改定位与可疑区域分析模块

篡改定位模块用于发现局部编辑、拼接、重绘或局部 AIGC 替换区域。模块将多尺度滑动窗口检测与特征统计异常分析相结合，并在全局判定与局部定位结果之上引入四象限分类器，区分"全图 AIGC 生成"与"局部篡改"两种不同场景。

2\.4\.1 多尺度滑动窗口检测

将图像分割为重叠的滑动窗口 patch，每个 patch 独立送入 Detector 推理，获得该区域属于 AIGC 伪造的概率。采用两种尺度（224×224 和 160×160，代码层支持扩展到 112×112）独立运行，步长比例 stride\_ratio 控制滑动密度——当前默认 0\.5，即相邻 patch 重叠 50%，可在覆盖密度与推理耗时之间取得平衡。多尺度结果取最大值融合，保留最强信号。

针对推理效率做了多项优化。包括：

- 大图降采样：最长边超过 500px 时先缩至 500px 分析，分析后上采样回原尺寸，大幅减少 patch 总数。

- 批量推理：patch 按 batch\_size=32 合批送入 GPU，减少 kernel launch 开销。

- 置信度过滤：fake\_prob 低于 0\.35 的 patch 直接置零，同时保留一份过滤前的原始分数`_raw_score_map`，供后续局部篡改检测风险评分使用。

- 右下角覆盖：滑动窗口可能遗漏右下角边缘，额外补充一个锚定右下角的 patch。

2\.4\.2 特征统计异常分析

利用 Detector 提供的 stage2 中间层特征（384 通道 × 14×14），计算每个通道在 14×14 空间上的方差作为异常权重，再用该权重加权原始特征得到空间异常图：

$\alpha_c = \frac{\text{Var}(F_c)}{\max_k \text{Var}(F_k)}, \quad S_{\text{feat}}(i,j) = \sum_{c=1}^{C} \alpha_c \cdot |F_c(i,j)|$

其中 $F_c$ 为第 $c$个通道的 14×14 特征图，$C=384$，经双线性上采样和高斯平滑后得到与图像同尺寸的可疑度图。该方法的优势在于单次前向传播、速度极快（\<1ms），且 14×14 空间分辨率能够捕捉局部纹理异常。

2\.4\.3 统一可疑度图生成

PatchAnalyzer 和 FeatureStatsAnalyzer 的输出按加权求和融合为统一可疑度图：

$S(i,j) = 0.4 \cdot S_{\text{patch}}(i,j) + 0.6 \cdot S_{\text{feat}}(i,j)$

feature 权重大于 patch，因为特征统计方法对全局篡改（全图 AIGC）更敏感且无滑动窗口的块状伪影。归一化后进入后处理流水线，最终输出的每个 bbox 附带 `patch_fake_prob` 字段——从 confidence\_floor 过滤前的原始 patch\_score 中提取该区域均值，确保局部篡改场景下即使全局 fake\_prob 很低，可疑区域也能获得反映局部真实威胁程度的风险分。

2\.4\.4 四象限局部篡改分类

在全局判定与局部定位结果之上构建四象限分类器，产生四种独立的局部证据类型。该分类器不改变 Detector 输出的全局标签：

|全局判定|局部定位|篡改类型|输出标签|含义|
|---|---|---|---|---|
|real|无 bbox|confirmed\_real|real|全图真且局部无异常，双重确认|
|real|有 bbox|local\_tamper|real|全局模型判真但 patch 级发现异常，保留证据分歧并提示复核|
|fake|无 bbox|full\_aigc|fake|全图假，伪影均匀分布，无明显热点|
|fake|有 bbox|full\_aigc\_hotspots|fake|全图假且某些区域伪影更集中，标注重点区域|

对于 local\_tamper，其设计依据是：全局 softmax 与 patch 级滑窗观察的是不同尺度的证据。小面积异常可能在全图表征中被稀释，也可能来自定位分支的误报，因此系统仅仅将其标记为局部异常证据，并要求人工结合原图、热力图和可疑区域复核。

每个 bbox 区域的局部风险分基于 patch 级 fake\_prob 独立计算：

$R_{\text{local}} = 0.5 \cdot p_{\text{patch}} + 0.5 \cdot \min\left(\frac{A_{\text{bbox}}}{A_{\text{total}}} \times 10,\ 1.0\right)$

其中 $p_{\text{patch}}$ 为该 bbox 区域内所有 patch 的原始 fake\_prob 均值`_raw_score_map`，$A_{\text{bbox}}$ 和 $A_{\text{total}}$ 分别为 bbox 面积与图像总面积。局部风险分由两大维度共同决定：区域本身的 AI 伪造置信度，代表局部纹理伪影强度；篡改区域占整张图的面积比例，代表篡改影响范围。与传统方案使用全局 fake\_prob 不同，patch 级分数确保局部篡改场景下，bbox 仍能获得合理的局部风险分，不受全局低分拖累。

2\.4\.5 全局风险评分

采用五维度加权综合评估整张图像的审核优先级：

$R_{\text{global}} = \sum_{k=1}^{5} w_k \cdot d_k$

其中 $d_k$ 为各维度分数，$w_k$ 为对应权重，$\sum w_k = 1$。

检测置信度（$w=0.30$）直接使用上游 MambaOut 模型输出的 fake\_prob。伪影强度（$w=0.25$）综合热力图的最大响应和平均响应：$0.6 \cdot \max(H) + 0.4 \cdot \overline{H}$，前者捕捉最强伪影信号，后者反映整体覆盖程度。篡改面积比（$w=0.25$）为所有 bbox 面积之和与图像总面积的比值。区域数量（$w=0.10$）采用对数归一化 $\log_2(n+1) / \log_2(6)$，5 个以上可疑区域时该项封顶 1\.0。一致性（$w=0.10$）计算热力图高分区域（前 25% 像素）与篡改掩膜高分区域的 IoU 重叠度——重叠度高说明 Grad\-CAM 解释与定位模块发现相互印证，增强可信度；重叠度低则提示两者可能存在分歧。

|维度|权重|计算方式|
|---|---|---|
|检测置信度|0\.30|fake\_prob|
|伪影强度|0\.25|0\.6 × heatmap\_max \+ 0\.4 × heatmap\_mean|
|篡改面积比|0\.25|Σ bbox\_area / \(W × H\)|
|区域数量|0\.10|log₂\(n\+1\) / log₂\(6\)|
|一致性|0\.10|IoU\(热力图高分区域, 掩膜高分区域\)|

风险等级分为三档：low \[0, 0\.35\) 绿色、medium \[0\.35, 0\.70\) 橙色、high \[0\.70, 1\.0\] 红色。当前生产配置仍保留这组可解释边界；在 Facebook 派生平衡集上另行得到 medium 候选边界 0.3947 和 high 候选边界 0.4232，并以未参与选阈值的留出集报告结果。候选边界尚未替换生产配置，也不能外推到其他来源。

2\.5 多证据风险融合模块

融合模块已实现为 2\.4\.5 所述的五维加权评分：输入全部来自上游合同字段（检测置信度、热力图统计、定位掩膜与 bbox 列表），输出 `risk_score`、低/中/高 `risk_level` 与报告化原因，用于排定人工复核优先级；融合结论不改写 Detector 的全局标签。当前等级边界为可解释的固定规则，其数据驱动校准结果与外推边界见 3\.4。

2\.6 平台实现与交互流程

TraceGuard 当前使用同一 `ExplanationPipeline` 支撑 Web、FastAPI、CLI 与批量分析入口。浏览器工作台由 `web/` 提供，正式接口包括健康检查、配置读取、单图分析和批量分析；CLI 与批量脚本复用相同检测器和解释管线，不在前端或脚本中重新计算真伪结论。

Web 最小闭环包括：用户上传图片，系统调用检测模型，展示真伪概率、风险等级、可解释图和定位结果，并生成单图检测报告。批量入口用于实验阶段导出 CSV/JSON 结果。

# 第三章 作品测试与分析

3\.1 测试目标与总体方案

在进入逐项实验之前,先给出检测器的能力小结,便于读者带着能力预期理解后续实验及其边界。其一,**零样本超监管识别**:面向危害最大的超监管高危内容,检测器在从未见过该域数据的零样本条件下,9 个生成器 Fake Recall 均落在 94.0%–100.0%、总体达 98.84%(表 3\.6)。其二,**真图低误伤**:跨生成器盲测中 Real Recall 稳定保持 99.80%(表 3\.1),超监管零样本评测中真图假阳率仅 1.4%,高召回未以牺牲真图判别为代价。其三,**可信路由**:全局 `label` 仅由 Detector 产生,局部证据不静默覆盖全局结论,证据退化或冲突时系统触发人工复核而非过度自信改判;风险校准在留出集上复核策略 F1 达 0.9877、高风险边界 F1 达 1.0(见 3\.4)。这些能力构成后续实验的读取框架:实验既验证这些能力,也诚实标定它们在难例生成器与有损压缩下的边界。

测试围绕两个核心问题展开:**其一,检测器能否识别未知生成器生成的图像**(跨生成器识别能力);**其二,同一图像经过社交媒体重编码后,检测证据还能保留多少**(传播鲁棒性)。这两个问题正对应作品的叙事主线——高危 AIGC 与普通内容共享同一条社交媒体传播链,而压缩恰恰最能摧毁伪造证据,因此传播链末端的第三方审核必须同时具备跨域识别与传播鲁棒两种能力,并在证据退化时转入人工复核。

跨生成器盲测使用平衡 real/fake 测试集;传播实验使用固定 `sample_id` 的 Original、Facebook、WeChat、Weibo 成对图像;派生扰动实验在此基础上用确定性图像处理拆解"是哪一种处理在摧毁证据"。所有预测均由同一份固定检测权重和统一 `Detector` 接口产生,逐样本结果与运行记录随复现材料一并归档,保证可追溯与可复现。

3\.2 数据来源与指标定义

**数据集说明**

跨生成器测试使用 GenImage\[3\] 的 ADM、BigGAN、Glide、Midjourney、SD14、SD15、VQDM 与 Wukong 八个生成器子集，每类 1000 张 fake 图像。每组同时使用同一来源的 1000 张 real 图像形成 1:1 平衡测试。

社交媒体成对实验使用上述 8000 张 GenImage fake 图像及其 Facebook、WeChat、Weibo 传播后版本，共形成 32000 个唯一预测键。平台分类实验另使用三个 `test_eachfake_500_real500` 归档，每个平台包含 500 张 real 与 4500 张 fake 图像。派生扰动实验对同一 8000 张 GenImage fake 图像施加确定性图像处理（jpeg75、jpeg50、resize50、screenshot），并额外抽取 1000 张 real 图像做重采样假阳对照。GenImage 官方数据采用 CC BY-NC-SA 4.0 并附加非商业使用条款\[8\]；平台派生归档与本地派生归档仅保存在本地、不随代码发布，其下载链路仍待补齐。

**指标定义**

- **Accuracy**：正确分类样本占全部样本的比例。
- **Macro F1**：real 与 fake 两类 F1 分数的算术平均。
- **ROC AUC**：以 `fake_prob` 为 fake 正类分数计算的 ROC 曲线下面积。
- **Real Recall / Fake Recall**：real、fake 两类分别被正确识别的比例。
- **Fake Recall 保持率**：传播（或扰动）后 Fake Recall 与 Original Fake Recall 的比值，衡量相对退化程度。
- **成对概率变化**：同一 `sample_id` 传播（或扰动）后 `fake_prob` 减去其 Original 值，在样本上取均值。

跨生成器盲测、社交媒体成对实验与平台分类实验均使用同一份固定检测权重，运行于单张消费级 GPU（RTX 4060 Laptop）。派生扰动实验使用同一份固定检测权重，该实验在另一硬件环境独立复跑，其 Original Fake Recall（59.54%）与 GPU 结果（59.55%）一致，跨环境复现稳定。实验所用权重、样本哈希与运行记录随复现材料一并归档。

**3\.3\.3 派生扰动深入分析（表 3\.4、表 3\.5）【新增】**

Facebook/WeChat/Weibo 三条真实链路是黑箱的复合处理，无法直接指认是哪一步在摧毁证据。为此，本作品在同一批 8000 张 GenImage fake 图像上施加**确定性派生扰动**——JPEG 质量 75、JPEG 质量 50、下采样至 50% 再放大（resize50）、以及屏幕截图链路（screenshot），逐条拆解退化来源，并对 resize50 出现的反常升高单独做真图对照。

表 3\.4 · 派生扰动 Fake Recall 保持率（GenImage 全量 8000 样本 × 5 条件，同一份固定检测权重，40000 条预测 0 失败）

| 条件 | Fake Recall | 保持率 |
|---|---:|---:|
| Original | 59.54% | 100.0% |
| jpeg75 | 9.91% | 16.65% |
| jpeg50 | 2.34% | 3.93% |
| resize50 | 82.13% | 137.94% † |

† resize50 的高保持率为“降采样再放大”引入的重采样偏置所致的**假象，非鲁棒性收益**，真图假阳检查见结论 2 与表 3.5。

**结论 1：JPEG 量化是主要的证据破坏者。** 仅一次质量 75 的 JPEG 重压缩就把 Fake Recall 从 59.54% 拉低到 9.91%（保持率 16.65%），质量 50 时更跌至 2.34%（保持率 3.93%）；screenshot 链路（含二次压缩与缩放）也把保持率压到 40.96%。这与 3.3.2 中 Facebook 的剧烈退化（保持率 36.398%）互相印证——真实社交平台链路普遍包含强 JPEG 重压缩，正是 Facebook 成为退化最严重平台的直接原因。JPEG 量化因此被确认为传播鲁棒性的首要威胁面。

**结论 2：resize50 的表面升高是假象，本质是重采样偏置而非鲁棒性收益。** 逐生成器结果显示，resize50 把几乎所有生成器的 Fake Recall 拉到约 0.83–0.89，与其原始 recall 高低无关：**VQDM 从 0.131 被抬到 0.834，达 6.37 倍**，ADM 从 0.37 抬到 0.866（2.34 倍）。这种“无差别抬升”提示模型是把“降采样再放大”本身读作了生成伪影，而非真的识别出更多伪造证据。为验证这一点，对 1000 张真图做同样处理的假阳检查（表 3.5）。

表 3\.5 · resize50 真图假阳检查（1000 张真图，同一份固定检测权重）

| 变体 | 真图假阳率(FP) | mean fake_prob |
|---|---:|---:|
| 原图-真图 | 1.50% | 0.096 |
| resize50-真图 | 17.50% | 0.245 |

**解释边界（必须说明）：** resize50 把真图假阳率从 1.50% 抬高到 17.50%（约 11.7 倍），1000 张真图中出现 160 例 real→fake 的单向翻转、0 例反向翻转，平均 fake_prob 从 0.096 升到 0.245。这证明“降采样再放大”被检测器普遍读作生成伪影，**不分真假地整体抬高 fake 预测**。因此 resize50 在表 3.4 中 Fake Recall “升至 137.94%” **并非传播鲁棒性收益，不能作为鲁棒性的正面证据**——它抬高的是全局 fake 倾向而非判别能力。这一发现恰好印证了系统的核心主张：证据在传播中退化或被扰动扭曲时，检测器不应给出过度自信结论，而应将可疑样本转交人工复核。

# 第四章 创新性说明

4\.1 超监管高危场景的零样本识别力与真图判别

本作品的核心识别能力体现在社会危害最大的超监管高危内容(战争、军事、武器类场景)上:检测器在从未见过该域数据的零样本条件下,对 9 个生成器的 Fake Recall 均落在 94.0%–100.0%、总体达 98.84%(表 3\.6),而 real 图像的假阳率仅 1.4%——高召回并非以牺牲真图判别力为代价。这一真图判别力在跨生成器盲测中同样稳健:Real Recall 稳定保持 99.80%(表 3\.1),几乎不冤枉真图。更进一步,超监管域的零样本识别力相对通用 GenImage 测试集的总体 Fake Recall(59.55%)不降反升,说明检测器学到的生成器伪影判据能够跨内容域迁移到高危场景,而这类高危内容恰是现有检测方法未专门覆盖的盲区\[9\]。

**能力边界。** 该超监管测试集的生成器构成与 GenImage 不同(含 CycleGAN、DALLE、Flux、LatentDM,不含 VQDM、ADM 等难例),绝对数值不宜与 GenImage 逐项对比,此处仅主张"零样本识别能力可跨内容域迁移"这一方向性结论。通用盲测中,当前权重对 VQDM 与 ADM 等难例的 Fake Recall 分别仅为 13.20% 和 37.00%,识别力在难例生成器上仍有欠缺;超监管高危内容在遭遇有损压缩后同样会大幅退化(见 4\.3)。在上述边界之内,检测器对未压缩超监管高危内容仍保持稳定的识别能力。

4\.2 三层输出合同与可信路由:证据退化即转人工

本作品将"证据退化时不给出过度自信结论"设计为系统的核心能力,而非事后附加的免责说明。系统始终保持"全局判断、局部证据、融合结论"三层输出合同:全局 `label` 与 `fake_prob` 仅由 Detector 产生,Grad\-CAM 热力图与局部定位作为并行证据分支,局部证据可以质疑全局判断、标记可疑区域,但不得静默覆盖全局结论,冲突本身即是转人工的信号。风险融合以真伪概率、热力图、篡改定位与文件处理信息的五维加权(见 2\.4\.5)输出统一的低/中/高风险等级与报告化解释,减少单一模型分数在复杂场景中的误导;在 Facebook 派生平衡集的分层留出校准上,人工复核策略的留出集 F1 达 0.9877、高风险边界 F1 达 1.0(见 3\.4),说明融合分数可支撑审核分流。融合结论不改写 Detector 的全局标签。

可信路由在证据退化时尤为重要:当同一图像的伪造概率因传播压缩由 0.967 跌到 0.018 时,系统据此触发人工复核,而非静默改判为 real(图 3\.3)。**该阈值候选仅支持本次来源、权重和划分下的取值,未替换生产配置,也不能外推到其他平台或未见域。**

4\.3 平台的自我诊断能力:传播链末端证据退化的系统度量

本作品把"传播链末端的证据退化"从一个附带观察升格为可复现的系统评测,这是区别于既有检测工作的核心创新点。该评测衡量的并非单纯的性能下降,而是平台能否判别自身结论何时不再可靠——即平台的自我诊断能力。评测以同图成对协议为基础:对固定的 8000 张 GenImage fake 图像,分别运行 Original、Facebook、WeChat、Weibo 四种条件,以同一 `sample_id` 追踪同一图像在传播前后的 `fake_prob` 与检出变化,从而直接度量"证据被削弱了多少",而非仅仅比较两个不同数据集上的分类精度。结果显示 Facebook 条件下 Fake Recall 由 59.55% 降至 21.675%、平均伪造概率下降 0.3162(表 3\.2),且逐生成器保持率差异悬殊(BigGAN、VQDM、ADM、Glide 仅 2.3%--11.8%,Midjourney、Wukong 达 63.5%--78.7%),说明传播鲁棒性必须同时看绝对检出率与相对保持率。

在此基础上,本作品用**确定性派生扰动**进一步拆解"是哪种处理在摧毁证据":对全量样本施加 jpeg75、jpeg50、resize50、screenshot 五条件,定位出 JPEG 量化是主要的证据破坏者(jpeg75 保持率仅 16.65%、jpeg50 仅 3.93%,表 3\.4),与 Facebook 传播退化互证(Facebook 链路含强 JPEG 重压缩)。针对 resize50 表面上"保持率升至 137.94%"的反常,进一步用真图假阳检查证伪:resize50 把 1000 张真图的假阳率从 1.50% 抬到 17.50%(表 3\.5),证明"降采样再放大"被检测器普遍读作生成伪影、不分真假地整体抬高 fake 预测,因此这并非鲁棒性收益,不作为传播鲁棒性的正面证据。正是系统能够量化并识别出这种证据退化与扰动扭曲,才能在证据不可信时把可疑样本精准分流到人工复核,而非给出过度自信的结论。相较于既有工作多止于原始图像或单次 JPEG 压缩这类孤立扰动的评测口径,本作品在真实社交平台传播链末端给出了证据退化的系统度量与成因拆解,使系统的性能主张与真实审核工况对齐,并直接支撑"证据退化即转人工复核"的处置策略。

4\.4 去全局池化的门控卷积骨干

传统骨干网络末端的全局平均池化会不可逆地压缩空间伪影信息。本作品选取 MambaOut 门控 CNN 并针对 AIGC 图像检测适配:移除末端全局平均池化,将最后一阶段特征保留为 2×2 空间网格并展平为 2304 维密集特征,再经瓶颈层降至 256 维,在保留微观伪影空间结构的同时控制后续域对齐的计算维度。与直接使用 ResNet/ViT 全局特征的常规做法\[1\-2\]相比,该设计以保留空间证据为首要目标。

4\.5 多核 MMD 的无监督域自适应机制

引入 5 核高斯 MK\-MMD,在再生核希尔伯特空间对齐源域与目标域的特征分布,目标域无需任何标注。与仅靠数据增广提升泛化的做法相比,该机制显式建模并度量多尺度的域间分布差异。骨干去全局池化改造与 MK\-MMD 无监督域自适应的分项量化消融(各自的独立贡献)列入后续工作,本报告不提前将其写为已复核结果。

4\.6 可解释取证与报告化闭环

系统将检测分数、热力图、可疑区域、解释文本与报告导出整合于同一流程(证据见图 3\.3 与表 3\.9),使检测结果可复核、可沟通、可归档。与仅输出分类分数的检测器\[1\-2\]和仅做归因可视化的方法\[7\]相比,本作品将解释产物纳入审核处置链路,提升了 AIGC 检测从算法结果到安全处置证据的可用性。

4\.7 轻量化部署与平台化工程闭环

TraceGuard 具备真部署能力。MambaOut\-Small 骨干约 45.44M 参数,已在单张 RTX 4060 Laptop GPU 上完成真实 Web 闭环:固定 BigGAN 样例的单图 API 分析耗时约 0.56 秒;12 请求、并发度 3 的短时并发烟测 12/12 成功,中位延迟 0.520 秒、P95 2.075 秒(表 3\.9),支持单卡审核工作台部署。工程上采用模块化结构,四个入口(Web UI、API、CLI 与批量分析)复用同一 ExplanationPipeline 与统一响应合同,将跨域检测、解释展示、篡改定位与报告导出组织为同一平台闭环,避免不同入口产生相互矛盾的真伪结论;179 项自动化回归保障各入口行为一致性。

# 第五章 总结

TraceGuard 面向 AIGC 图像安全中的检测、解释、取证与报告需求,构建跨域检测、可解释分析、篡改定位、风险融合与平台展示一体化的作品框架。它交付的不是"一个强检测器",而是"一个在证据退化时依然可信的审核平台":在社会危害最大的超监管高危内容上零样本识别力总体达 98.84%、真图假阳率仅 1.4%(表 3\.6),跨生成器盲测 Real Recall 稳定保持 99.80%(表 3\.1),三层输出合同保证证据退化即转人工而非静默改判,且已在单张消费级 GPU 上完成真实 Web 闭环。其叙事主线是:超监管高危内容是危害最大的伪造,却与普通内容共享同一条最能摧毁伪造证据的社交媒体传播链,因此传播链末端需要可解释、传播鲁棒、证据退化即转人工的第三方审核。

**已完成进展**:系统已实现 MambaOut\-Small 全局检测、Stage2 Grad\-CAM、patch 与特征统计融合定位、五维风险评分、中文解释以及 Web/API/CLI/HTML 报告入口,并以 179 项自动化回归保障入口行为一致。当前权重已完成八生成器平衡盲测、三类社交平台传播成对实验、确定性派生扰动实验与超监管高危内容的零样本行为边界评价。作为核心贡献,一方面在头号高危场景上验证了检测器的零样本识别力与可信路由能力;另一方面,传播前后成对实验系统量化了 Facebook 压缩导致的显著性能退化(Fake Recall 由 59.55% 降至 21.675%),派生扰动进一步定位 JPEG 量化为主要证据破坏者(jpeg75 保持率 16.65%),并以真图假阳检查证伪了 resize 的表面鲁棒性假象。这一证据退化的系统度量体现了平台的自我诊断能力;它同时表明,不能仅用类别不平衡平台集上的较高 Accuracy 代替传播链末端的鲁棒性分析。

系统最终保持"全局判断、局部证据、融合结论"三层输出。全局 `label` 与 `fake_prob` 仅由 Detector 产生;热力图与局部定位作为并行证据分支;风险模块依据多维证据给出审核优先级与人工复核建议。当前来源的定位边界评价与留出风险校准已写入报告;定位分支的低像素重合、clean 误报与单一传播来源限制了其外推范围,而这恰恰印证了系统的核心处置主张——在证据退化、来源单一或证据冲突时明确转人工复核,而不给出过度自信的结论。此外,超监管零样本评测的生成器构成较 GenImage 更易(无 VQDM/ADM),其绝对值不与 GenImage 逐项比较,仅支持"零样本识别可跨内容域迁移"的方向性结论;骨干与域自适应机制的分项消融列为后续工作。
